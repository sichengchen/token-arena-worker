# Web I18n and Theme Switching Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add complete `next-intl`-based `zh`/`en` routing plus persisted light/dark/system theme switching to the existing web app.

**Architecture:** Introduce locale-prefixed App Router routes backed by `next-intl` request/routing config, then expand the existing preference model so signed-in users persist locale/theme in the database while signed-out users use cookies/local storage. A lightweight custom theme provider handles first-paint theme resolution, and locale-aware navigation/switchers are integrated into auth and usage surfaces.

**Tech Stack:** Next.js 16 App Router, next-intl, React 19, Prisma, Vitest, Biome, Tailwind CSS 4
